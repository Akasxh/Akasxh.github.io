Hierarchical Attention with Speculative Token Eviction

Flash attention is fast because it is IO aware, but it still computes attention over every single KV block. For long contexts, most of those blocks contribute almost nothing to the output. The attention distribution is sparse. The question is whether you can figure out which blocks matter without actually computing full attention over all of them. That is what HASTE does.

HASTE is a 3 level cascade filter that sits inside the online softmax loop. Each level is cheaper than the next, and most blocks get rejected early. Only the blocks that survive all three stages get the full dot product treatment. The result: 57 to 65% of KV blocks are skipped entirely, with less than 1% accuracy loss.

The Three Stages

Stage 1 is a Cauchy Schwarz norm gate. This is the cheapest possible filter because it requires zero K loads. The idea is simple: the dot product Q*K is bounded by ||Q|| * ||K||. If the upper bound on the attention score is already below a threshold, there is no way this block matters. We precompute and cache ||K|| norms per block, so the gate is just a scalar multiply and compare. Most irrelevant blocks die here.

Stage 2 is a low dimensional sketch probe. For blocks that pass the norm gate, we project Q and K into a much smaller subspace, d' = 16 versus the full d = 128. The sketch gives a rough estimate of the actual attention score at 8x less compute than the full dot product. Blocks whose sketch score falls below threshold get evicted here.

Stage 3 is full attention, computed only for the surviving blocks. By this point, we have typically eliminated 57 to 65% of all blocks through the first two stages, so the full attention computation covers a much smaller working set.

Temporal Coherence with EMA

Attention patterns in autoregressive generation are not random. Consecutive tokens tend to attend to similar regions of the KV cache. HASTE exploits this with an exponential moving average (EMA) over block importance scores. If a block was important for the last few tokens, it is likely important for the next one too. The EMA provides a warm prior that biases the cascade: blocks with high historical importance get a lower eviction threshold, making them harder to skip. Blocks that have been consistently irrelevant get a higher threshold, making them easier to evict early.

This temporal coherence mechanism is what separates HASTE from static sparsity patterns. The sparsity adapts per token, per head, per layer. Some heads are sparse, some are dense. Some tokens attend broadly, some focus narrowly. The EMA lets HASTE track these dynamics without any overhead beyond a single scalar update per block per step.

Error Bounded Correction

Speculative eviction is inherently approximate. You are making decisions about block importance based on cheap proxies, and sometimes those proxies are wrong. A block might have a low norm gate score but contain a few keys that are critically important. To handle this, HASTE runs periodic correction passes that recompute attention over recently evicted blocks and measure the actual approximation error.

If the error exceeds a configurable bound, the correction pass is promoted: the block gets re included in the active set, and its EMA score is updated to prevent future false evictions. This gives HASTE a formal error guarantee. You can set the maximum allowable approximation error, and the correction mechanism ensures it is never exceeded over a rolling window.

Results and Takeaways

Across various sequence lengths and model sizes, HASTE consistently achieves 57 to 65% sparsity with under 1% degradation in downstream accuracy. The cascade overhead (norm gate + sketch probe) is small enough that net wall clock time decreases once sequences are long enough for the memory bandwidth savings to dominate. The sweet spot is around 4K tokens and above, where the KV cache is large enough that skipping 60% of it matters.

The key insight is that attention sparsity is not a fixed pattern you can hardcode. It varies per head, per layer, per token. Any scheme that tries to impose a static mask (local windows, stride patterns, fixed top k) will either be too aggressive for dense heads or too conservative for sparse ones. The cascade approach lets each block prove its own relevance at minimal cost, and the EMA ensures the system adapts over time without explicit re planning.
